A Robottriónknak edzésre van szüksége a jövőben rájuk váró kalandokra valamint a kincsvadászatra (mindenkinek szüksége van arany érintkezőkre) való felkészülés érdekében. István épített egy egyszerű piramismodellt. Most robotjaink aranyvadász futamedzést fognak végezni időre. A csúcsról kezdik el és aranyat gyűjtenek minden szobában, amelyen áthaladnak. Minden szobában kiválasztanak vagy egy balra vagy egy jobbra vezető irányt és így ereszkednek lefelé a következő szintre. Az eredmények kiértékeléséhez Istvánnak tudnia kell a maximálisan összegyűjthető aranymennyiséget.

Képzeljünk el egy n-es értéksorokból (tuple) álló n-es értéksort, amelyben az első tömb egy számot tartalmaz, a következők, – egy számmal többet, mint az előző. Az ilyen n-es értéksor egy háromszögre fog hasonlítani. Önnek egy függvényt kell írnia, amely segít Istvánnak meghatározni a legnagyobb aranymennyiséget a piramis csúcsától az alapjáig tartó leggazdaságosabb útvonalon. Minden útvonal lefelé és balra/jobbra irányuló lépésekből állhat.

Megjegyzés: tekintse a lefelé és balra irányuló lépést, mint egy lépést a következő sorba, hogy a sorbéli indexet nem változtatja meg, és a lefelé és jobbra irányuló lépést, mint egy lépést a következő sorba, hogy a sorbéli index értéke növekszik eggyel. Járjon el óvatosan a rekurzióval, mivel annak alkalmazása lassúbb megoldást eredményez!

Bemeneti adatok: a piramis n-es értéksorokból álló (tuple) n-es sorokként megadva. Minden n-es értéksor egész számokat (int) tartalmaz.

Kimeneti adatok: az egy futam alatt összegyűjthető legnagyobb aranymennyiség egész számként (int).

Példák:

count_gold((
    (1,),
    (2, 3),
    (3, 3, 1),
    (3, 1, 5, 4),
    (3, 1, 3, 1, 3),
    (2, 2, 2, 2, 2, 2),
    (5, 6, 4, 5, 6, 4, 3)
)) == 23
count_gold((
    (1,),
    (2, 1),
    (1, 2, 1),
    (1, 2, 1, 1),
    (1, 2, 1, 1, 1),
    (1, 2, 1, 1, 1, 1),
    (1, 2, 1, 1, 1, 1, 9)
)) == 15
count_gold((
    (9,),
    (2, 2),
    (3, 3, 3),
    (4, 4, 4, 4)
)) == 18

Alkalmazása: Ez a klasszikus feladat segíthet a dinamikus programozás elsajátításában. Továbbá hasznos lehet számos szélsőérték-feladat megoldásában.

Előfeltételek: 0 < len(pyramid) ≤ 20
all(all(0 < x < 10 for x in row) for row in pyramid)
